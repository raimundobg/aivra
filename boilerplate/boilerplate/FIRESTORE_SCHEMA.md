# Firestore Schema Documentation

## Overview
This document describes the Firestore database structure for the Aivra nutrition platform.

## Collections

### `/users/{uid}`
Main user document. Contains user metadata and role information.

**Fields:**
- `uid` (string): User's authentication UID (document ID)
- `email` (string): User's email address
- `role` (string): User role - 'patient' or 'nutritionist'
- `assignedNutritionistUid` (string, optional): UID of the assigned nutritionist (patients only). Used in Firestore security rules to enforce access control.

**Security Rules:**
- Patients can read/write their own document
- Assigned nutritionists can read/update patient documents
- Unassigned nutritionists cannot access patient data
- Users cannot delete their own documents

**Example:**
```json
{
  "uid": "patient-123",
  "email": "patient@example.com",
  "role": "patient",
  "assignedNutritionistUid": "nutritionist-456"
}
```

### `/users/{patientUid}/intake/{doc}` (Subcollection)
Patient's intake form data. Filled during onboarding.

**Fields:**
- `edad` (number): Age in years
- `sexo` (string): 'M' or 'F'
- `altura` (number): Height in cm
- `peso` (number): Weight in kg
- `actividadFisica` (string): Activity level ('sedentario', 'ligero', 'moderado', 'intenso', 'muy_intenso')
- `nivelEducacion` (string): Education level
- `ocupacion` (string): Occupation
- `medicamentos` (array of strings): List of medications
- `suplementos` (array of strings): List of supplements
- `cirugias` (array of strings): History of surgeries
- `antecedenteFamiliar` (array of strings): Family medical history
- `alergias` (array of strings): Known allergies
- `diagnosticosOtros` (array of strings): Other medical diagnoses
- `desayuno` (array of AlimentoSeleccionado): Breakfast foods
- `colacion_am` (array of AlimentoSeleccionado): Morning snack foods
- `almuerzo` (array of AlimentoSeleccionado): Lunch foods
- `colacion_pm` (array of AlimentoSeleccionado): Afternoon snack foods
- `cena` (array of AlimentoSeleccionado): Dinner foods

**AlimentoSeleccionado structure:**
```json
{
  "alimento": "Arroz blanco",
  "medida_casera": "1 taza",
  "kcal_porcion": 130,
  "proteinas_porcion": 2.7,
  "carbohidratos_porcion": 28,
  "lipidos_porcion": 0.3,
  "porciones": 1,
  "kcal_total": 130,
  "proteinas_total": 2.7,
  "carbohidratos_total": 28,
  "lipidos_total": 0.3
}
```

**Security Rules:**
- Patients can read/write their own intake
- Assigned nutritionists can read only
- Deletion is blocked

### `/users/{patientUid}/r24/{dateString}` (Subcollection)
24-hour dietary recall for a specific date (YYYY-MM-DD format).

**Fields:**
- `meals` (object): Object with meal keys as properties
  - `desayuno` (array of AlimentoSeleccionado)
  - `colacion_am` (array of AlimentoSeleccionado)
  - `almuerzo` (array of AlimentoSeleccionado)
  - `colacion_pm` (array of AlimentoSeleccionado)
  - `cena` (array of AlimentoSeleccionado)
- `savedAt` (timestamp): Last save timestamp

**Security Rules:**
- Patients can read/write their own R24
- Assigned nutritionists can read/write to analyze and provide feedback
- Deletion is blocked

---

### `/users/{patientUid}/pauta/{doc}` (Subcollection)
Generated dietary plan (pauta) generated by the AI or nutritionist.

**Fields:**
- `titulo` (string): Plan title
- `descripcion` (string): Plan description
- `macros` (object):
  - `calorias` (number): Target calories
  - `proteinas` (number): Target protein in grams
  - `carbos` (number): Target carbs in grams
  - `grasas` (number): Target fats in grams
- `desayuno` (array of meal items)
- `colacion_am` (array of meal items)
- `almuerzo` (array of meal items)
- `colacion_pm` (array of meal items)
- `cena` (array of meal items)
- `generadoPor` (string): 'nutricionista' or 'sistema' (IA)
- `generadoAt` (timestamp): Creation timestamp

**Security Rules:**
- Patients can read/write their own pauta (needed for IA self-generation during onboarding)
- Assigned nutritionists can read/write
- Deletion is blocked

---

### `/users/{patientUid}/ficha/{doc}` (Subcollection)
Clinical record/assessment document. Written by nutritionist.

**Fields:**
- `diagnosticoNutricional` (string): Nutritional diagnosis
- `observaciones` (string): Clinical observations
- `recomendaciones` (array of strings): Recommendations
- `creadoAt` (timestamp): Creation timestamp
- `actualizadoAt` (timestamp): Last update timestamp

**Security Rules:**
- Patients can read
- Only assigned nutritionists can write
- Deletion is blocked

---

### `/users/{patientUid}/habits/{date}` (Subcollection)
Patient's daily habits tracking (sleep, stress, exercise, etc.).

**Fields:**
- `fecha` (string): Date in YYYY-MM-DD format
- `horasDormidas` (number): Hours of sleep
- `calidadSueno` (number): Sleep quality (1-10)
- `nivelEstres` (number): Stress level (1-10)
- `ejercicio` (string): Type of exercise
- `duracionEjercicio` (number): Duration in minutes
- `estado` (string): General state/mood

**Security Rules:**
- Patients can read/write their own habits
- Assigned nutritionists can read only
- Deletion is blocked

---

### `/users/{patientUid}/mediciones/{doc}` (Subcollection)
Body composition measurements (weight, measurements, body fat, etc.).

**Fields:**
- `fecha` (string): Date of measurement
- `peso` (number): Weight in kg
- `cintura` (number): Waist circumference in cm
- `cadera` (number): Hip circumference in cm
- `pecho` (number): Chest circumference in cm
- `porcentajeGrasa` (number): Body fat percentage (calculated via Durnin-Womersley)

**Security Rules:**
- Patients can read/write their own measurements
- Assigned nutritionists can read only
- Deletion is blocked

---

### `/users/{patientUid}/menu/{doc}` (Subcollection)
Weekly menu generated by IA or nutritionist.

**Fields:**
- `semana` (string): Week identifier
- `menuDia1` through `menuDia7` (objects): Daily menus with meals
- `generadoAt` (timestamp)

**Security Rules:**
- Patients can read/write
- Assigned nutritionists can read only
- Deletion is blocked

---

### `/users/{patientUid}/recetas/{doc}` (Subcollection)
Recipes generated or curated for the patient.

**Fields:**
- `nombre` (string): Recipe name
- `ingredientes` (array): List of ingredients with quantities
- `instrucciones` (string): Preparation instructions
- `tiempoPreparacion` (number): Time in minutes
- `porciones` (number): Number of servings
- `macros` (object): Macro information per serving

**Security Rules:**
- Patients can read/write
- Assigned nutritionists can read only
- Deletion is blocked

---

### `/conversations/{patientUid}/messages/{msgId}` (Subcollection)
Chat messages between patient and nutritionist.

**Fields:**
- `de` (string): Sender UID
- `texto` (string): Message text
- `timestamp` (timestamp): Message send time
- `leido` (boolean): Whether patient has read the message

**Security Rules:**
- Both patient and assigned nutritionist can read and create messages
- Messages cannot be updated or deleted (immutable)

---

### `/_emailThrottle/{doc}` (Internal)
Rate limiting storage for email notifications (Cloud Functions only).

**Security Rules:**
- Blocked from all user access
- Cloud Functions have full access via service account

## Security Rule Principles

1. **Nutritionist Assignment**: All nutritionist access is gated by `assignedNutritionistUid` field in the patient's user document. This prevents unauthorized nutritionists from accessing patient data.

2. **Null Guards**: All role-checking functions include null guards to prevent crashes if a user document is missing.

3. **Immutable Clinical Records**: Deletes are blocked on all clinical/personal data. Updates are only allowed where necessary (pauta generation, habit tracking, etc.).

4. **Patient Primacy**: Patients always have access to their own data and can write to forms/records that feed into the system.

5. **Read-Only for Nutritionists**: Most data (intake, habits, mediciones) is read-only for nutritionists to prevent accidental modifications.
